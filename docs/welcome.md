# Welcome to Why Grief Matters CMS

This is the content management studio for [whygriefmatters.org](https://whygriefmatters.org). Here you can create, edit, and organize grief support resources that help people navigate loss and bereavement.

> **Important:** The website is statically generated — your changes appear after the next site build, not immediately when you publish. Builds can be triggered manually via the **Website Management** tool

## How the Studio Is Organized

The Studio sidebar groups content into four sections:

1. **Singletons** — One-of-a-kind documents. Currently this is just the Organization profile, which controls site-wide branding, logos, and legal text.

2. **Documents** — Core content types that structure the site:

   - **Categories** — The hierarchical topic tree that organizes all content
   - **Populations** — Audience segments for filtering resources
   - **Content Blocks** — Reusable page sections made of text, images, and links
   - **Content Groups** — Page-level containers that arrange content blocks
   - **Crisis Resources** — Emergency support services with contact information
   - **Websites** — Source/attribution records for internet resources
   - **People** — Team member profiles
   - **Person Groups** — Named groups of people (e.g., "Board of Directors")
   - **Image Collections** — Groups of images used as fallbacks or for other specific purposes

3. **Internet Resources** — 18 types of curated external links (articles, books, podcasts, videos, apps, and more). This is the bulk of the site's content.

4. **Documentation** — These help pages (you're reading one now).

## What Your Content Does

Everything you create or edit in this Studio feeds into the whygriefmatters.org website. The key things to understand:

- **Categories** define the site's navigation structure. Resources tagged with categories appear on those category pages.
- **Populations** enable filtered views of content for specific audiences.
- **Content Blocks and Content Groups** let you compose custom page layouts or change content on existing pages.
- **Internet Resources** are the core content — curated links to external grief support materials.

Field descriptions in the Studio explain what each field does and any constraints — these docs focus on how things work _together_ and how your content appears on the website.

New to Sanity? See the [Sanity documentation](https://www.sanity.io/docs) for general concepts like documents, drafts, publishing, and rich text editing.
